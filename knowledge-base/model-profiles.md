# Model Profiles

## short

Use for concise summaries, command output interpretation, and simple file
lookups. Prefer low reasoning.

## standard

Use for ordinary repo edits, documentation updates, and focused code work.
Prefer medium reasoning.

## deep

Use for hardware risk analysis, architecture, source-backed research, protocol
design, and multi-step implementation. Prefer high or xhigh reasoning when the
quality gain justifies latency and cost.

## validation

Use for independent review of assumptions, source coverage, CI failures, safety
claims, and release gates.
