Public-safe quality gates

01

Generated artifact policy

The public site is assembled from an explicit allowlist in scripts/build_github_pages.py. The generated package publishes curated site files, selected Markdown, named WebP backplates, the static admin HMI demo, and public-file-manifest.json.

Read deployment model
02

Manifest, link, and hash audit

scripts/audit_public_manifest.py verifies manifest entries, file sizes, SHA-256 hashes, blocked source paths, image allowlist membership, local HTML references, and CSS asset links.

Open manifest
03

Page smoke checks

scripts/smoke_github_pages.py checks the landing page, visual blueprint page, this quality page, and the static admin HMI demo for required titles, required markers, and broken local references.

Open admin HMI demo
04

Safe-core host tests

Host-side tests cover relay safety gates, public API channel contracts, storage failure paths, and XBee frame parser rejection behavior without requiring a connected ESP32 or live relay module.

Read task model
05

XBee read-only boundary

Published XBee work stays at passive discovery and fixed read-only AT-query planning. XBee setting writes, API transmit frames, and ESP32 carrier DIN/DOUT wiring remain blocked.

Read XBee proof
06

Explicit non-coverage

The public artifact does not validate live ESP32 wiring, power rails, boot-pin effects, relay trigger polarity, relay input current, isolation, load-side wiring, mains wiring, live firmware flashing, live GPIO, MicroSD mount behavior, or TFT wiring.

Read power gates
Private evidence stays private. The public package excludes `.agents/` records, raw bench records, private uploads, screenshots, copied vendor PDFs, bulky binaries, and any unallowlisted generated media.
